INHERITANCE

Create new classes (derived classes) from existing classes (base classes)

The derived class inherits the variables and functions of the base class and adds additional ones!

Provides the ability to re-use existing code

Types of Inheritance

  • Single level inheritance
  • Multi-level inheritance
  • Hierarchical inheritance
  • Hybrid inheritance
  • Multiple inheritance

Single Level Inheritance

Single base class & a Single derived class 

public class Mobile  
{  
       //Properties  
       //Methods  
}  
  
public class Samsung : Mobile  
{  
       //Properties  
       //Methods  
}  

Multilevel Inheritance

In Multilevel inheritance, there is more than one single level of derivation 

public class Mobile  
{  
       //Properties  
       //Methods  
}  
  
public class Samsung : Mobile  
{  
       //Properties  
       //Methods  
}  

public class SamsungS5 : Samsung  
{  
       //Properties  
       //Methods  
}

Hierarchical Inheritance

Multiple derived class would be extended from base class 

public class Mobile
{
	//Properties  
	//Methods  
}
public class Samsumg : Mobile
{
	//Properties  
	//Methods  
}
public class Nokia : Mobile
{
	//Properties  
	//Methods  
}

Hybrid Inheritance

Single, Multilevel, & Hierarchal inheritance all together construct a Hybrid inheritance.

Multiple Inheritance

Multiple inheritance where derived class will extend from multiple base classes.

Polymorphism

  • Refers to the ability of a variable, function or object to take on multiple forms
  • Polymorphism can be static or dynamic.
    • In static polymorphism, the response to a function is determined at the compile time.
    • In dynamic polymorphism, it is decided at run-time.

Static Polymorphism

The mechanism of linking a function with an object during compile time is called early binding. It is also called static binding. C# provides two techniques to implement static polymorphism. They are:

  • Function Overloading
  • Operator Overloading

Function Overloading

  • Can have multiple definitions for the same function name in the same scope.
  • The definition of the function must differ from each other by the types and/or the number of arguments in the argument list. 
public class Logger
{
	public void Log(string message)
	{
		//Some code
	}

	public void Log(string message, Severity severity)
	{
		//Some code
	}
}

Dynamic Polymorphism

  • Can be achieved using virtual methods
  •  Have methods having identical signatures present in both the base and the derived classes
  • Virtual method is one that is declared as virtual in the base class and can allow the subclasses of the type to override the virtual method 
public class Logger
{
	public virtual void Log(string message)
	{
		Console.WriteLine("Inside the Log method of the base class Logger");
	}
}
public class FileLogger : Logger
{
	public override void Log(string message)
	{
		Console.WriteLine("Inside the Log method of the FileLogger class");
	}
}
static void Main(string[] args)
{
	Logger logger = null;

	if (args.Length > 1)
		logger = new Logger();
	else
		logger = new FileLogger();

	logger.Log("Hello World!");

	Console.ReadKey();
}

Interfaces

  • A syntactical contract that all the classes inheriting the interface should follow.
  • Defines the 'what' part of the syntactical contract.
  • Deriving classes define the 'how' part of the syntactical contract.
  • Define properties, and methods, which are the members of the interface
  • Interfaces contain only the declaration of the members.
  • It is the responsibility of the deriving class to define the members.
